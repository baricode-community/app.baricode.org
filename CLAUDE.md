## Project Overview

**Baricode Community Platform** — an Indonesian IT community platform for learning, sharing, and collaboration. Built with Laravel 13 + Livewire/Volt + Filament admin panels.

## Commands

```bash
# Full dev environment (server + queue + logs + vite hot reload)
composer run dev

# Run tests
composer run test

# Run a single test file
php artisan test tests/Feature/ExampleTest.php

# Format code (Laravel Pint)
./vendor/bin/pint

# First-time setup
composer run setup
```

## Architecture

### Route Organization

Routes are split by feature domain under `routes/web/`:
- `auth.php` — login, register, OAuth (Google), password reset
- `home.php` — landing page, profile, short link redirect, how-to-learn guide
- `dashboard.php` — authenticated dashboard (home, settings, analytics, memes, fun, onboarding)
- `family.php` — community member directory
- `cheatsheet.php` — cheatsheet CRUD
- `tracker.php` — daily commit tracker
- `timeline.php` — timelines
- `repohub.php` — repository hub
- `lms.php` — LMS courses, lessons, enrollment, lesson/category progress, quizzes
- `fun.php` — memes
- `mentoring.php` — mentoring programs and enrollments

`routes/api.php` exposes REST endpoints for courses and timelines, plus the MCP server at `/api/mcp`.

### Admin Panel (Filament v5)

Single panel at `/admin` (`app/Filament/`). Resources cover all major models: Users, Courses, Lessons, Enrollments, CategoryProgress, Quizzes, Memes, Mentoring, OnboardingTasks, RepoHub, ShortLinks, Timelines, CheatSheets, HowToLearn.

### Model Namespacing

Models are grouped by feature domain:
- `App\Models\Quiz\` — Quiz, Question, Option, QuizAttempt, QuizResult
- `App\Models\LMS\` — Course, Lesson, CourseCategory, Enrollment, LessonProgress, CategoryProgress, HowToLearn, YoutubeList
- `App\Models\Fun\` — Meme, MemeVote
- `App\Models\Mentoring\` — MentoringProgram, MentoringEnrollment, MentoringSession
- `App\Models\Onboarding\` — OnboardingTask, OnboardingTaskCompletion
- `App\Models\RepoHub\` — RepoHub, RepoHubTag
- Top-level: User, Timeline, ShortLink, CheatSheet, CheatSheetCategory, DailyCommitTracker, Meet, UserMeet, ProgressJournal

### Frontend Stack

- **Livewire Volt** — page components in `resources/views/pages/` (single-file components)
- **Livewire Flux** — UI component library
- **Tailwind CSS v4** via Vite
- Blade anonymous components under `resources/views/components/`

### Authentication

- **Web routes**: Session-based via Laravel Fortify (includes 2FA support)
- **API routes**: Token-based via Laravel Sanctum (`auth:sanctum` middleware)
- **OAuth**: Google via Laravel Socialite (`GoogleAuthController`)
- **Authorization**: Spatie Permission package for roles/permissions; `App\UserRole` enum defines `admin`/`user` cases

### Key Infrastructure

- **Queue**: Database driver — run via `php artisan queue:listen`
- **Cache/Session**: Database driver
- **File Storage**: AWS S3 via `league/flysystem-aws-s3-v3`
- **API Docs**: Auto-generated by `dedoc/scramble`
- **MCP Server**: `App\MCP\PlatformMcpServer` exposes platform data (stats, commits, courses, quizzes) as AI tools via Laravel MCP at `/api/mcp`
- **Performance**: Laravel Octane available

### Services and Supporting Layers

- `App\Services\` — `CourseImportExportService`, `QuizImportExportService`, `LMS/EnrollmentService`
- `App\Observers\` — `ProgressJournalObserver`
- `App\Policies\Mentoring\MentoringEnrollmentPolicy`
- `App\Listeners\General\Security\` — login/logout audit logging
- `App\Actions\Fortify\` — Fortify action overrides (password reset, profile update, etc.)
- `App\Enums\` — shared enums

### Testing

Uses **Pest** with SQLite in-memory for tests. Feature tests in `tests/Feature/`, unit tests in `tests/Unit/`. Test directories mirror the feature structure (Auth, DailyCommitTracker, Dashboard, Family, Fun, LMS, Mentoring, Quiz, RepoHub, Settings, Onboarding).

### CI/CD

GitHub Actions:
- `.github/workflows/lint.yml` — PHP Pint linting
- `.github/workflows/tests.yml` — Pest test suite (PHP 8.4, Node 22); requires `FLUX_USERNAME` and `FLUX_LICENSE_KEY` secrets for Flux UI composer credentials

## Feature Notes

### LMS Enrollment Flow
Enrollment is managed by `EnrollmentService`. Students can enroll/unenroll per course; unenroll requires a cancellable request step. Lesson and category progress are tracked separately.

### Mentoring
Programs → Enrollments (UUID-keyed) → Sessions. Enrollment requires auth; `MentoringEnrollmentPolicy` gates access. Enrollment UUID is used as the route key.

### Onboarding
`OnboardingTask` records with slug-based routing. Users toggle completion via `OnboardingController@toggle`. Progress is tracked through `OnboardingTaskCompletion`.

### Family Directory
Route model binding uses `{user:username}`. Controllers select only safe fields at query level — email and phone are never exposed.

### CheatSheet
Full CRUD with auth-gated create/edit/delete. Browseable publicly. Belongs to `CheatSheetCategory`.
