## Project Overview

**Baricode Community Platform** ‚Äî an Indonesian IT community platform for learning, sharing, and collaboration. Built with Laravel 12 + Livewire/Volt + Filament admin panels.

## Commands

```bash
# Full dev environment (server + queue + logs + vite hot reload)
composer run dev

# Run tests
composer run test

# Run a single test file
php artisan test tests/Feature/ExampleTest.php

# Format code (Laravel Pint)
./vendor/bin/pint

# First-time setup
composer run setup
```

## Architecture

### Route Organization

Routes are split by feature domain under `routes/web/`:
- `general.php` ‚Äî public/general routes
- `lms.php` ‚Äî Learning Management System routes
- `fun.php` ‚Äî fun/entertainment features (memes, etc.)
- `whatsapp.php` ‚Äî WhatsApp bot integration

### Admin Panels (Filament v5)

Two separate Filament panels:
- `/admin` ‚Äî main admin panel (`app/Filament/`)
- `/admin-lms` ‚Äî LMS-specific admin panel (`app/Filament/AdminLms/`)

Each panel has its own `PanelProvider` and `Resources/` directory for CRUD resource definitions.

### Model Namespacing

Models are grouped by feature domain:
- `App\Models\Quiz\` ‚Äî Quiz, Question, Option, QuizResult
- `App\Models\LMS\` ‚Äî Course, Lesson, CourseCategory
- `App\Models\Fun\` ‚Äî Meme, MemeVote

### Frontend Stack

- **Livewire Volt** ‚Äî page components in `resources/views/pages/` (single-file components)
- **Livewire Flux** ‚Äî UI component library
- **Tailwind CSS v4** via Vite

### Authentication

- **Web routes**: Session-based via Laravel Fortify (includes 2FA support)
- **API routes**: Token-based via Laravel Sanctum (`auth:sanctum` middleware)
- **OAuth**: Google via Laravel Socialite (`GoogleAuthController`)
- **Authorization**: Spatie Permission package for roles/permissions

### Key Infrastructure

- **Queue**: Database driver ‚Äî run via `php artisan queue:listen`
- **Cache/Session**: Database driver
- **File Storage**: AWS S3 via `league/flysystem-aws-s3-v3`
- **API Docs**: Auto-generated by `dedoc/scramble`
- **Performance**: Laravel Octane available

### Testing

Uses **Pest** with SQLite in-memory for tests. Feature tests in `tests/Feature/`, unit tests in `tests/Unit/`.

### CI/CD

GitHub Actions runs:
- `.github/workflows/lint.yml` ‚Äî PHP Pint linting
- `.github/workflows/tests.yml` ‚Äî Pest test suite (PHP 8.4, Node 22)

## Web Features Catalog

### Authentication & Settings (Protected Routes)
- **Settings** ‚Äî User account settings (requires `auth` middleware)
  - Profile settings (`/settings/profile`)
  - Password change (`/settings/password`)
  - Appearance settings (`/settings/appearance`)
  - Two-factor authentication (`/settings/two-factor`, conditional on Fortify config)
- **Google OAuth** ‚Äî Social login via Google (guest routes)
  - Redirect to Google (`/auth/google`)
  - Google callback handler (`/auth/google/callback`)
